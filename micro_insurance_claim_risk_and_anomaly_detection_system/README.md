# Micro Insurance Claim Risk and Anomaly Detection System

AIML capstone project: a decision-support system that flags suspicious micro-insurance claims for human review using machine learning, explainable reason codes, and review routing.

---

## Problem Statement

Insurance operations teams cannot manually inspect every claim in detail. This system:

- Scores each claim for fraud/anomaly risk
- Explains why a claim was flagged (reason codes)
- Recommends the next human-review action

The system **does not auto-deny claims**. It helps investigators prioritize suspicious cases.

## Dataset / Reference Source

**Dataset:** [Insurance Fraud Detection (Kaggle)](https://www.kaggle.com/datasets/arpan129/insurance-fraud-detection)

| Property | Value |
|---|---|
| Records | 1,000 claims |
| Features | 39 columns |
| Target | `fraud_reported` (`Y` / `N`) |
| Fraud cases | 247 (24.7%) |
| Non-fraud cases | 753 (75.3%) |

### Bundled in this repo

After clone, prepare the working CSV with:

> A pre-trained model is already included, so you can run the demo immediately. Use `prepare_data.py` before retraining or EDA.

## AI/ML Component

| Component | Description |
|---|---|
| Model | Voting ensemble (Random Forest + Gradient Boosting) |
| Imbalance handling | SMOTE on training data |
| Risk score | Fraud probability (0–1) from `predict_proba` |
| Risk tiers | HIGH (≥0.70), MEDIUM (0.40–0.69), LOW (<0.40) |
| Reason codes | Rule-based explanations (claim amount, police report, witnesses, claim ratios, model confidence) |
| Review actions | `AUTO_APPROVE`, `REVIEW_RECOMMENDED`, `ESCALATE_TO_FRAUD_TEAM` |

The ML model detects complex fraud patterns. Rule-based reason codes make outputs understandable for human reviewers.

## How to Run

### 1. Install

```bash
git clone <your-repo-url>
cd micro_insurance_claim_risk_and_anomaly_detection_system
python3.12 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Prepare data (from bundled archive)

```bash
python src/prepare_data.py
```

### 3. Quick demo (pre-trained model included)

```bash
python src/predict.py --sample
streamlit run app/app.py
```

### 4. Retrain (optional)

### 5. Full pipeline

## Results and Insights

**Test-set performance (Voting Ensemble + SMOTE, threshold = 0.26):**

| Metric | Value |
|---|---|
| Recall | 85.7% |
| Precision | 62.7% |
| F1 | 72.4% |
| ROC-AUC | 85.6% |
| Accuracy | 84.0% |

**Confusion matrix (test set):**

|  | Predicted safe | Predicted fraud |
|---|---|---|
| Actually safe | 126 | 25 |
| Actually fraud | 7 | 42 |

**Key insights:**

1. Fraud claims often show higher amounts and unusual injury/property/vehicle ratios.
2. Missing police reports and zero witnesses are common risk signals.
3. Recall was prioritized so investigators catch most suspicious claims.
4. The voting ensemble outperformed logistic regression and standalone random forest on F1.

## Limitations

- Small public dataset (1,000 rows); not production-ready
- Schema-specific — new fields require retraining
- Model assists reviewers; does not replace human judgment
- Some false positives expected (~37% of flagged claims may be non-fraud)
